# دليل واجهة المستخدم وتجربة المستخدم (UI/UX) - لوحة تحكم مالك التطبيق (Super Admin Dashboard)

هذا الدليل مفصل بالكامل لمالك النظام (الـ Super Admin) الذي يقوم ببيع النظام كـ (SaaS - Software as a Service) لشركات المقاولات الأخرى. الهدف من اللوحة هو البساطة، ووضوح الأرقام، وسهولة إدارة المشتركين.

---

## 1. الهيكل العام للتصميم (Layout Structure)
يفضل استخدام نموذج اللوحات الإدارية الكلاسيكية والعملية:
- **الشريط الجانبي (Sidebar)** على اليمين (في حال كانت الواجهة عربية)، يحتوي على روابط التنقل الأساسية بأيقونات واضحة.
- **الشريط العلوي (Header)** يحتوي على:
  - اسم المالك أو صورة حسابه.
  - إشعارات التنبيه (مثال: شركة اقتربت من تجاوز مساحة التخزين الخاصة بها).
  - زر تسجيل الخروج (Logout).
- **المنطقة المركزية (Main Content)** لعرض الجداول والإحصائيات وتفاصيل الشاشات.
- **شكل التصميم (Theme)**: يُفضل استخدام ألوان هادئة واحترافية (أزرق داكن Dark Blue كألوان أساسية، مع خلفيات رمادية فاتحة لتمييز البطاقات Cards).

---

## 2. الشاشات المطلوبة ومحتوياتها بالتفصيل

### الشاشة الأولى: الرئيسية (Overview / Dashboard)
**الهدف**: إعطاء المالك نظرة طائرة على أداء النظام واستهلاك الموارد.
**المحتويات**:
1. **بطاقات إحصائية (Stat Cards)** في أعلى الشاشة:
   - **إجمالي الشركات النشطة (Active Companies)**: رقم كبير مع نسبة النمو.
   - **إجمالي الشركات المعلقة (Suspended Companies)**: لمعرفة الاشتراكات المتأخرة الدفع.
   - **إجمالي المساحة المستخدمة (Total Storage Used)**: مثلاً `150 GB / 2 TB`، لمعرفة مقدار الحمل على سيرفرات المالك.
   - **أحدث شركة مسجلة**: اسم أحدث شركة انضمت للنظام.
2. **رسم بياني (Chart)**: 
   - يوضح نمو اشتراكات الشركات خلال الأشهر الأخيرة، أو يوضح أعلى الشركات استهلاكاً للمساحة.
3. **لائحة (Table) مختصرة**: "أحدث الشركات المضافة" تحتوي اسم الشركة والتاريخ لمعاينة سريعة.

---

### الشاشة الثانية: إدارة الشركات (Companies Management)
**الهدف**: التحكم الكامل باشتراكات المقاولين ومراقبة سحبهم للبيانات.
**المحتويات**:
1. **جدول عرض بيانات الشركات (Data Table)**:
   - أعمدة الجدول: (ID، شعار واسم الشركة، حالة الاشتراك، رقم التواصل، مساحة الخزن المستهلكة مقابل المسموح بها، تاريخ الانضمام، الإجراءات).
   - توفير فلتر وخانة بحث للبحث عن شركة معينة بالاسم أو رقم الهاتف.
   - **علامات ملونة (Badges) في عمود الحالة**: أخضر (Active)، أحمر (Suspended). وعلامات للخزن: أحمر إذا تجاوزت الشركة 90% من استهلاك الخزن.
2. **نافذة/صفحة: إضافة شركة جديدة (Create Company)**:
   - **القسم الأول (بيانات الشركة)**: حقل الاسم المستعار، رقم التواصل، البريد الإلكتروني، إمكانية رفع الشعار (Logo)، تحديد مساحة التخزين (بالميجا أو الجيجا).
   - **القسم الثاني (إنشاء حساب مدير الشركة - Admin User)**: إنشاء المدير الذي سوف يُعطى حسابه للمقاول ليدخل نظامه. (الاسم، اسم المستخدم Username، كلمة المرور Password).
3. **شاشة/نافذة تعديل شركة مجدولة (Edit / Suspend Company)**:
   - زر لتعديل مساحة التخزين (عندما ترغب الشركة بشراء مساحة إضافية).
   - زر التبديل السريع (Toggle Switch) لتغيير حالة الشركة من Active إلى Suspended للتعليق (في حالة عدم تسديد اشتراكهم المالي لك). أي شركة متعلقة، يتوقف نظامهم ومديريهم وعمالهم عن الدخول للتطبيق فوراً.

---

### الشاشة الثالثة: إدارة العملات (Currencies Management)
**الهدف**: بما أنك تدعم تعدد العملات وصرفها، يحتاج النظام لقاعدة مركزية.
**المحتويات**:
1. جدول لعرض العملات (مثلاً: دولار، دينار عراقي).
2. زر إضافة عملة جديدة (Code: EUR, Symbol: €).

---

### الشاشة الرابعة: تقارير مساحة التخزين (Storage & Media Logs)
**الهدف**: بما أن تطبيق المقاولات سيفعل نظام رفع (الوصولات، الفواتير، صور الإنجاز للمواقع)، سيقوم التطبيق باستهلاك مساحة الهاردسك الخاصة بسيرفرك، يجب عليك رؤية من يستهلكه.
**المحتويات**:
1. **قائمة (List) أو (Progress Bars)**:
   - يظهر كل شركة وشريط يوضح كم مساحة استهلكت.
   - مثال: "شركة مقاولات بغداد" - استهلكت `800 MB` من أصل `1 GB` المتاحة لهم (يظهر الشريط بالبرتقالي للتحذير).
2. إمكانية إرسال إشعار تحذيري (في التحديثات القادمة) للشركات التي أوشكت على إنهاء سعتها.

---

### الشاشة الخامسة: الإعدادات والأمان للمالك (Super Admin Profile & Settings)
**الهدف**: إدارة الحساب المالك نفسه.
**المحتويات**:
1. تغيير كلمة مرور (المالك نفسه - Super Admin).
2. إعدادات اسم واسم مستخدم (Username) مالك النظام.
3. التحديثات وقواعد البيانات (للمستقبل).

---

## 3. تجربة المستخدم والتفاعلات (Micro-interactions)
- **Toast Notifications (تنبيهات عائمة)**: عندما يقوم المالك بإيقاف (Suspend) شركة، يجب أن يظهر له إشعار صغير أخضر سريع في أسفل الزاوية "تم تعليق عمل الشركة بنجاح".
- **Modal Confirmations (نوافذ تأكيدية)**: لا تدع المالك يغلق شركة هكذا بالخطأ!، يجب ظهور نافذة: "هل أنت متأكد من رغبتك في إيقاف عمل شركة (العمران)؟ سيؤدي ذلك لمنع مديريها من الدخول".
- **Loading States (حالات التحميل)**: عند الضغط على إضافة شركة، اجعل الزر يظهر أيقونة الدوران (Spinner) وأوقف تفاعل باقي الأزرار حتى يكتمل المعالجة التي تولد حساب المقاول الجديد وترسل لك استجابة الـ Token لضمان عدم إنشاء مرتين.
- **Empty States (الحالات الفارغة)**: في حال لم تكن هناك شركات بعد، يفضل ظهور صورة توضيحية خفيفة (Illustration) تحمل نص "لم تقم بتسجيل أي شركة مقاولات بعد، أضف عميلك الأول!".

---

## 4. المسارات البرمجية (API Routes & Integrations) للوحة التحكم
لربط الواجهات المذكورة أعلاه مع الخادم (Backend)، تحتاج لوحة تحكم المالك إلى برمجة واجهاتها على المسارات (Routes) التالية. تم توحيد الاستجابات (Responses) لتكون دائماً بالهيكل التالي: `{ status, message, data }`.

### 1- تسجيل الدخول (Login)
يسجل المالك (Super Admin) دخوله من خلال هذا المسار للحصول على التوكن.

**شروط الإدخال (Validation):**
- `username`: مطلوب (Required).
- `password`: مطلوب (Required).

```json
{
  "URL": "/api/login",
  "Method": "POST",
  "Request_Body": {
    "username": "superadmin",
    "password": "password"
  },
  "Response_200": {
    "status": true,
    "message": "تم تسجيل الدخول بنجاح.",
    "data": {
      "id": 1,
      "name": "Super Admin",
      "username": "superadmin",
      "email": "superadmin@example.com",
      "role": "SUPER_ADMIN",
      "company_id": null,
      "token": "1|jfdkjf88djskkdlds..."
    }
  },
  "Response_401_Error": {
    "status": false,
    "message": "بيانات الدخول غير صحيحة.",
    "data": null
  }
}
```

### 2- المستخدم الحالي (Current User)
يتم التحقق من أن التوكن المخزن لا يزال فعالاً.
```json
{
  "URL": "/api/user",
  "Method": "GET",
  "Headers": {
    "Authorization": "Bearer {token}"
  },
  "Response_200": {
    "status": true,
    "message": "تم جلب بيانات المستخدم بنجاح.",
    "data": {
      "id": 1,
      "name": "Super Admin",
      "username": "superadmin",
      "email": "superadmin@example.com",
      "role": "SUPER_ADMIN"
    }
  }
}
```

### 3- جلب جميع الشركات (Fetch Companies)
تملأ هذه الاستجابة "جدول عرض بيانات الشركات".
```json
{
  "URL": "/api/super-admin/companies",
  "Method": "GET",
  "Headers": {
    "Authorization": "Bearer {token}"
  },
  "Response_200": {
    "status": true,
    "message": "تم جلب الشركات بنجاح.",
    "data": [
      {
        "id": 1,
        "name": "شركة المهندس الذهبي",
        "status": "active",
        "logo_path": "companies/logos/example.png",
        "contact_number": "07712345678",
        "email": "info@golden-eng.com",
        "storage_limit_mb": 500.00,
        "storage_used_mb": 120.50,
        "users_count": 5,
        "projects_count": 2
      }
    ]
  }
}
```

### 4- إنشاء شركة جديدة وحساب للمدير (Create Company)
تُستدعى هذه الإضافة من واجهة "إضافة شركة جديدة".

**شروط الإدخال (Validation Requirements):**
- `name`: مطلوب، نص، أقصى طول 255 حرف.
- `contact_number`: اختياري، نص.
- `email`: اختياري، صيغة بريد إلكتروني صالحة.
- `storage_limit_mb`: اختياري، رقم لا يقل عن 0.
- `status`: مطلوب، يجب أن يكون بقيمة `active` أو `suspended`.
- `logo`: اختياري، ملف صورة، أقصى حجم 5120 كيلوبايت (5 ميجا).
- `admin_name`: مطلوب، نص، أقصى طول 255.
- `admin_username`: مطلوب، نص، يجب أن يكون فريداً (غير مكرر) في قاعدة البيانات (Unique).
- `admin_password`: مطلوب، نص، لا يقل عن 6 أحرف.

```json
{
  "URL": "/api/super-admin/companies",
  "Method": "POST",
  "Headers": {
    "Authorization": "Bearer {token}"
  },
  "Request_Body": {
    "name": "شركة المقاولات الحديثة",
    "contact_number": "07800000000",
    "email": "modern@company.com",
    "storage_limit_mb": 1024,
    "status": "active",
    "admin_name": "اسم مدير الشركة",
    "admin_username": "new_admin",
    "admin_password": "password123",
    "logo": "file (اختياري - فقط مع FormData)"
  },
  "Response_201": {
    "status": true,
    "message": "تم إنشاء الشركة بنجاح.",
    "data": {
      "company": { ... },
      "admin_user": { ... }
    }
  },
  "Response_422_Validation_Error": {
    "status": false,
    "message": "حدثت مشكلة في التحقق من صحة البيانات.",
    "data": null,
    "errors": {
      "admin_username": ["اسم المستخدم هذا مستخدم مسبقاً، يرجى اختيار اسم آخر."]
    }
  }
}
```

### 5- تحديث إعدادات الشركة أو تعليقها (Update/Suspend Company)
يتم استخدام هذا المسار من "نافذة تعديل شركة". يمكن إرسال حقل وتجاهل الآخر.

**شروط الإدخال (Validation Requirements):**
- `name`: اختياري، نص.
- `contact_number`: اختياري، نص.
- `email`: اختياري، إيميل.
- `storage_limit_mb`: اختياري، رقم.
- `status`: اختياري، يقبل فقط `active` أو `suspended`.
- `logo`: اختياري، ملف صورة.

```json
{
  "URL": "/api/super-admin/companies/{id}",
  "Method": "PUT",
  "Headers": {
    "Authorization": "Bearer {token}"
  },
  "Request_Body": {
    "status": "suspended",
    "storage_limit_mb": 2048
  },
  "Response_200": {
    "status": true,
    "message": "تم تحديث الشركة بنجاح.",
    "data": {
      "id": 2,
      "status": "suspended",
      "storage_limit_mb": 2048
    }
  }
}
```

### 6- تسجيل الخروج (Logout)
يُستخدم لإتلاف التوكن (Token) وحماية الحساب.
```json
{
  "URL": "/api/logout",
  "Method": "POST",
  "Headers": {
    "Authorization": "Bearer {token}"
  },
  "Response_200": {
    "status": true,
    "message": "تم تسجيل الخروج بنجاح.",
    "data": null
}
```

### 7- قطاع المعدات الثقيلة (Heavy Equipment)
هذا القطاع منفصل تماماً عن قطاع المقاولات، ويستخدم لإدارة الآليات وتأجيرها وإيراداتها.

#### أ. إضافة / جلب الآليات (Equipment)
**شروط الإدخال (Validation Requirements):**
- `name`: مطلوب، نص (مثال: كرين، شفل).
- `model`: اختياري، نص.
- `plate_number`: اختياري، نص (رقم اللوحة الشاصي).
- `status`: مطلوب (active, in_maintenance, out_of_service).

```json
{
  "URL": "/api/heavy-equipment/equipment",
  "Method": "POST",
  "Headers": {
    "Authorization": "Bearer {token}"
  },
  "Request_Body": {
    "name": "شفل كاتربلر",
    "model": "CAT 320D",
    "plate_number": "س ب 1234",
    "status": "active"
  },
  "Response_201": {
    "status": true,
    "message": "تم إضافة الآلية بنجاح.",
    "data": { ... }
  }
}
```

#### ب. إضافة / جلب السواق (Drivers)
**شروط الإدخال (Validation Requirements):**
- `name`: مطلوب، نص.
- `phone_number`: اختياري، نص.
- `salary_type`: مطلوب (monthly, percentage, per_operation).

```json
{
  "URL": "/api/heavy-equipment/drivers",
  "Method": "POST",
  "Request_Body": {
    "name": "أحمد محمود",
    "phone_number": "07700000000",
    "salary_type": "monthly"
  }
}
```

#### ج. تسجيل الإيرادات للآليات (Operations / Revenue)
**شروط الإدخال:**
- `equipment_id`: مطلوب (أيدي الآلية).
- `driver_id`: اختياري (أيدي السائق).
- `work_date`: مطلوب، تاريخ.
- `revenue_amount`: مطلوب، رقم.
- `currency_id`: مطلوب (أيدي العملة).

```json
{
  "URL": "/api/heavy-equipment/operations",
  "Method": "POST",
  "Request_Body": {
    "equipment_id": 1,
    "driver_id": 2,
    "work_location": "بغداد - مشروع المطار",
    "work_date": "2023-10-25",
    "revenue_amount": 150000,
    "currency_id": 1,
    "description": "إيجار لمدة 8 ساعات"
  }
}
```

#### د. تسجيل المصروفات التشغيلية (Expenses)
**شروط الإدخال:**
- `equipment_id`: مطلوب.
- `expense_category`: مطلوب (fuel, maintenance, etc).
- `cost_amount`: مطلوب.
- `currency_id`: مطلوب.
- `expense_date`: مطلوب.
- `invoice_image`: اختياري (صورة).

```json
{
  "URL": "/api/heavy-equipment/expenses",
  "Method": "POST",
  "Request_Body": {
    "equipment_id": 1,
    "expense_category": "fuel",
    "cost_amount": 50000,
    "currency_id": 1,
    "expense_date": "2023-10-25",
    "description": "تعبئة وقود ديزل لكامل الخزان"
  }
}
```
