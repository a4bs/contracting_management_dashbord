# توثيق النظام: الشاشات والواجهات البرمجية (API Documentation)

هذا التوثيق يوضح كيفية توزيع الصلاحيات، تصميم الشاشات لكل رتبة مستخدم، وتفاصيل المسارات البرمجية (Routes) المطلوبة للربط بين الواجهة الأمامية والخلفية.

---

## 1. شاشات مدير النظام (App Admin - Owner)
*هذا المستخدم يمتلك صلاحيات كاملة على مستوى النظام ككل.*

### الشاشات المطلوبة:
1. **لوحة التحكم الرئيسية (Dashboard)**:
   - إحصائيات عامة: عدد الشركات المتعاقدة، إجمالي الدخل، عدد المستخدمين الكلي.
   - تنبيهات النظام الحرجة.
2. **إدارة الشركات (Companies Management)**:
   - عرض، إضافة، تعديل، وحذف الشركات.
   - مراقبة استهلاك الشركات للموارد (Storage & Resource Index).
3. **إدارة المستخدمين العامين**:
   - إدارة حسابات الموظفين التابعين لإدارة التطبيق.

---

## 2. شاشات مدير الشركة (Company Admin)
*المسؤول عن إدارة عمليات شركته فقط.*

### الشاشات المطلوبة:
1. **نظرةcdمة (Analytics)**:
   - عدد المشاريع الجارية، عدد المعدات، إجمالي المصاريف والسلف.
2. **إدارة المشاريع (Projects)**:
   - قائمة المشاريع والعملاء.
   - إمكانية إضافة مشروع جديد (مع إنشاء عميل تلقائياً).
3. **إدارة الموارد البشرية (Staff & Drivers)**:
   - إضافة الموظفين والسائقين وربطهم بالمعدات.
4. **إدارة المعدات (Machinery)**:
   - إضافة المعدات ومراقبة عمليات الصيانة والمصاريف الخاصة بكل معدة.
5. **المعاملات المالية (Financials)**:
   - مراقبة سلف ومصاريف كل مشروع.
6. **مدير الملفات (Media Center)**:
   - رفع الوثائق، العقود، والصور.

---

## 3. شاشات العميل (Client)
*هذا المستخدم يرى فقط ما يخص مشاريعه.*

### الشاشات المطلوبة:
1. **مشاريعي (My Projects)**:
   - عرض حالة المشروع (Status).
2. **التقرير المالي (Financial Report)**:
   - عرض الدفعات والسحب الخاص بمشروعه فقط.
3. **التنبيهات (Alerts)**:
   - استلام تنبيهات عند حدوث أي إجراء في مشروعه.

---

## 4. شاشات السائق (Driver)
*يركز على المعدة المسؤولة عنه فقط.*

### الشاشات المطلوبة:
1. **المعدة الحالية (Assigned Machinery)**:
   - تفاصيل المعدة المسندة إليه.
2. **تسجيل العمليات (Log Operations)**:
   - تسجيل (وقود، صيانة، عمل يومي، إيراد).
   - إرفاق صور الوصولات.

---

# الملحق التقني: تفاصيل الـ API

| الأسلوب | المسار (Route) | البارامترات المطلوبة (Params) | الاستجابة المتوقعة (Response) |
| :--- | :--- | :--- | :--- |
| **POST** | `/api/login` | `username`, `password` | `token`, `user_data` (Role, Company) |
| **GET** | `/api/companies` | - | قائمة الشركات مع بيانات الاستهلاك |
| **POST** | `/api/projects` | `name`, `company_id`, `client_name` | بيانات المشروع + بيانات العميل المنشأ حديثاً |
| **GET** | `/api/projects/filter` | `name`, `status`, `client_id` | قائمة المشاريع المفلترة |
| **POST** | `/api/file-manager` | `file` (Multipart/form-data) | `url`, `file_name`, `file_size` |
| **POST** | `/api/machinery-operations`| `machinery_id`, `type`, `amount`, `attachment_url` | تأكيد الإضافة بنجاح |
| **GET** | `/api/system-alerts` | - | التنبيهات الخاصة بالمستخدم الحالي |
| **PATCH**| `/api/system-alerts/{id}/read`| `id` | تحديث حالة التنبيه إلى 'مقروء' |

---

### ملاحظات هامة للبرمجة:
- **Headers**: يجب إرسال `Authorization: Bearer {token}` في جميع الطلبات ما عدا اللوجن.
- **Filtering**: جميع مسارات الـ filter تستخدم `GET` وتدعم البحث الجزئي.
- **Storage**: الوصول للمرفقات يكون عبر `asset('storage/' . $path)`.
